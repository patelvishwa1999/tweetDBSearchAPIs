Functionalities:

Data Collection and Storage:

We have utilized the Twitter data provided by the professor

The data is stored in two separate data stores: one relational (PostgresQL) and one non-relational(Dynamodb).

We use efficient Python code(a Lambda Function) to process tweets one at a time, simulating real-time data processing.

Caching Mechanism:

Our project includes a LRU caching mechanism for storing frequently accessed data, such as popular hashtags, users, user details, tweet details.

The cache is implemented using a Python dictionary with eviction strategies to manage its size.

Cache Status page shows the data in cache for that moment to ensure the working of cache.

Search Application:

Our search application allows users to search tweets by hashtag and user.

After the search results, you can view meta-data such as tweet author, tweet time, and retweet count for each of the tweets.

Drill-down search features enable users to explore user deatils and other actions by the user author(tweets).

Top-Level Metrics:

We provide top users by 3 metrics to give users quick insights into the most active users
